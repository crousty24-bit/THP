# User Stories

1. As a junior developer, I want to get a list of test cases from a function so
   that I can understand which behaviors should be verified.

2. As a developer working on existing code, I want to generate regression tests
   before making a change so that I can reduce the risk of breaking existing
   behavior.

3. As a member of a small team, I want to choose the target test framework so
   that I can get a draft that matches the project's conventions.

4. As a developer, I want to read an explanation for each proposed test so that
   I can check whether the suggestion matches the expected behavior.

5. As a reviewer, I want to see the assumptions made by the AI so that I can
   quickly identify what must be confirmed or corrected.
