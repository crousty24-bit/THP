# Personas

## Persona 1: Lina, Junior Web Developer

Lina has recently joined a small web development team. She understands the
basics of unit testing, but she often struggles to decide which cases are worth
testing first.

Her needs:

- Identify the most important test cases quickly.
- Learn how to structure readable tests.
- Understand edge cases without asking for help every time.
- Avoid copying generated tests without understanding them.

How the tool helps:

The unit test generator gives Lina a first draft and explains why each test
exists. She can use the result as a learning support, then adjust the tests
before committing them.

## Persona 2: Marc, Full-Stack Developer on a Legacy Project

Marc works on a codebase with limited test coverage. Before modifying an old
module, he wants to add a few tests to protect existing behavior.

His needs:

- Understand what an existing function appears to do.
- Generate regression tests before refactoring.
- Spot unclear assumptions in the code.
- Save time without lowering code quality.

How the tool helps:

The tool analyzes the selected code and proposes tests for normal behavior,
boundary conditions, and possible error cases. It also lists assumptions that
Marc must verify manually before trusting the generated tests.
