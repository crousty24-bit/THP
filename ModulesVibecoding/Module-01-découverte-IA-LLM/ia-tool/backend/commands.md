# Rails API setup commands

These commands prepare the future Rails backend for Vibecoding IA Tool.
They are written for documentation and training purposes only: they are not
executed automatically in this project.

## 1. Create the Rails API application

```bash
rails new backend --api
```

Creates a new Rails application named `backend` in API-only mode. API mode keeps
Rails focused on controllers, models, routes, JSON responses, and database
logic, without generating the full server-rendered view stack.

## 2. Enter the Rails application folder

```bash
cd backend
```

Moves into the future Rails application before running generators and database
commands.

## 3. Generate the main API resource

```bash
rails generate scaffold TestDraft source_code:text framework:string proposed_tests:text draft_code:text explanations:text assumptions:text
```

Generates the main resource for this project: a `TestDraft`.

- `source_code:text`: code submitted by the developer.
- `framework:string`: target test framework, such as Jest, Vitest, or RSpec.
- `proposed_tests:text`: list of suggested test cases.
- `draft_code:text`: generated test code draft.
- `explanations:text`: explanation of why each test matters.
- `assumptions:text`: assumptions that the developer must verify.

## 4. Create and migrate the database

```bash
rails db:create
rails db:migrate
```

Creates the development database and applies the migration generated by the
scaffold.

## Complete command sequence

```bash
rails new backend --api
cd backend
rails generate scaffold TestDraft source_code:text framework:string proposed_tests:text draft_code:text explanations:text assumptions:text
rails db:create
rails db:migrate
```
